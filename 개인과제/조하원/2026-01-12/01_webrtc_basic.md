# 🧩 WEBRTC(Web Real-Time Communication)

## ✔️ WebRTC란?

웹 브라우저끼리 플러그인 없이도 실시간으로 영상·음성·데이터를 주고받게 해주는 기술
- 화상통화
- 화면 공유
- 음성 통화
- 실시간 협업 서비스
- 원격 제어

이런 기능을 Chrome, Firefox, Safari만 있으면 바로 구현할 수 있게 해 줌

## ✔️ WebRTC가 왜 특별한가?
### 1️⃣ 초저지연(LOW LATENCY)

- 데이터가 서버를 거치지 않고 브라우저 → 브라우저로 직접 전달됨
- 그래서 딜레이가 거의 없음 (원격제어·게임에 필수)

### 2️⃣ P2P 기반
- Peer-to-Peer 방식이라 중간에서 중계 서버를 반드시 활용하지 않아도 됨
(단, NAT 환경에서는 TURN 서버 필요 — 이건 기술적으로 필요, 확정된 내용)

### 3️⃣ 영상/음성 코덱 지원 내장
- VP8, VP9, H.264 (영상)
- Opus, G.711 (음성)
브라우저가 기본 지원함 → 별도 설치 필요 없음.

## ✔️ WebRTC 세 가지 주요 구성 요소
### 1. getUserMedia()

- 카메라/마이크 접근
- “기기에서 영상·음성 데이터를 가져오는 역할”

### 2. RTCPeerConnection

- 실제로 영상·음성을 복붙하듯 연결하는 핵심 객체
- “브라우저끼리 직접 연결(P2P)을 만들어주는 엔진”

### 3. RTCDataChannel

- 일, 채팅 메시지 등 비디오/오디오 말고 데이터 전송용
- “실시간 게임·텍스트·파일 전송 가능”
