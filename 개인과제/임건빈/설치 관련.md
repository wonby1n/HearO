**Node.js 버전 선택**
2026년 기준 가장 무난한 24 LTS 버전을 선택하자. 홀수보다는 짝수 버전이 보안과 안정성이 보장되니 실무와 학습용으로는 LTS가 답이다.

**NVM(Node Version Manager) 사용**
노드를 직접 깔지 말고 관리 도구를 쓰자. 버전 교체가 쉽고 삭제도 깔끔하다.

* Windows: nvm-windows 사용
* Mac/Linux: nvm 사용

**Node.js 설치 및 활성화**
터미널에서 명령어 몇 줄이면 끝난다.

* `nvm install lts` : 가장 안정적인 최신 LTS 버전 다운로드
* `nvm use lts` : 설치한 버전을 시스템에서 사용하도록 활성화
* `node -v` : 설치된 노드 버전 확인

**PowerShell 보안 정책 해결**
윈도우에서 npm 명령어가 안 먹히는 건 보안 정책 때문이다. 당황하지 말고 권한을 조정하자.

* `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` : 현재 사용자에게 스크립트 실행 권한 부여
* `RemoteSigned` : 내가 만든 건 허용하고 외부 파일은 서명된 것만 허용하는 합리적인 설정

**환경 세팅 마무리**
버전 확인까지 끝났다면 이제 개발 시작할 준비는 끝났다.

* `npm -v` : 패키지 매니저 실행 여부 최종 확인
* `Get-ExecutionPolicy -List` : 권한 설정이 제대로 반영됐는지 체크

**Node.js 설치 및 실행 문제**
터미널에서 'npm', 'node' 명령어를 인식하지 못하는 것은 프로그램이 없거나 경로(Path)가 잡히지 않은 상태임
오류 해결을 위해 환경 변수 확인보다 우선 설치 여부를 먼저 체크하자

**Node.js 설치 방식 선택**
직접 설치와 Docker 활용 중 무엇이 정답이라기보다 본인의 상황에 맞춰서 선택하자

직접 설치 (Local)

* 입문자에게 권장
* IDE(VS Code) 연동이 쉽고 실행 속도가 빠름
* 버전 관리가 필요하다면 NVM(Node Version Manager)을 사용하는 것이 국룰

Docker 활용 (Container)

* 내 PC 환경을 깨끗하게 유지하고 싶을 때 사용
* 배포 환경과 개발 환경을 완벽하게 일치시킬 때 유리
* 다만 설정이 복잡하고 초기 학습 비용이 발생함

**Docker 실행 오류 (Daemon)**
'failed to connect to the docker API' 에러는 도커 엔진이 꺼져 있다는 뜻임
먼저 Docker Desktop이 실행 중인지 확인하고 고래 아이콘의 상태를 살피자
안 된다면 WSL 2 설정이나 서비스 상태를 강제로 다시 시작해야 함

**Docker 내 npm install의 위치**
컨테이너 안에서 실행한 npm install은 내 컴퓨터(C 드라이브)가 아니라 컨테이너 내부 격리된 공간에 저장됨

* 컨테이너 내부 파일 시스템: 패키지가 실제로 존재하는 곳
* 호스트 PC(내 컴퓨터): 볼륨(Volume) 설정을 하지 않으면 보이지 않음
* 휘발성 특징: 컨테이너를 삭제하면 내부 패키지도 같이 사라지니 이미지를 빌드하거나 볼륨을 연결하자
